0. THE DISCIPLINE THIS ROADMAP ENFORCES
==================================================================
Quoting the Phase-1 definition (docs/Nidaa-Day12-Pilot-Definition.md):
  "This pilot does NOT test W-A, does NOT close OQ-1/OQ-2, does NOT generate
   G1/G2 humanitarian cases... A 'strengthen' here is necessary-but-not-sufficient
   for the Nidaa thesis."

Consequence: NO Phase-1 result (strengthen, weaken, or falsify) is allowed to be
described as humanitarian validation. Each phase is a HARD GATE: advance only on
evidence, never on optimism.

Also enforced (director, this session):
  "If A8 falsifies while A6b holds, stop building and write the negative result up."
  This is the exit condition that makes the work scientific rather than committed-
  A falsified A8 = a successful pilot, not a failure.

PHASE 0 ADDITION (this update): a new Phase 0 — Instrument Pilot — is inserted
BEFORE Phase 1. It is strictly instrument validation (operational rehearsal),
quarantined from all hypothesis inference by a standing rule below. It does not
alter the Phase 1 design or any assumption; it only de-risks running it.

==================================================================
1. THE FOUR PHASES
==================================================================
PHASE MAPPING (per director, this update):
  P0 = validate the INSTRUMENT    |    P1 = test the MECHANISM (proxy)
  P2 = test the MECHANISM inside a REAL organization
  P3 = evaluate HUMANITARIAN transferability

PHASE 0 — INSTRUMENT PILOT (NOW, before Phase 1)
  Spec: docs/Nidaa-Phase0-Instrument-Pilot-Spec.md
  Design: a dedicated synthetic rehearsal team (N=6–8, knowingly experimental; no
          humanitarian / NGO framing) runs the SAME templates (A–E) + 5 workflow
          actions as Phase 1 on a stand-in coordination scenario.
  Purpose: INSTRUMENT VALIDATION ONLY. Verify instructions are understandable,
           steward workload practical, templates usable, event logs practical,
           pressure protocol functional, surveys understandable, metrics
           collectable, and that the operational process breaks nowhere — BEFORE
           real participants.
  STANDING GOVERNANCE RULE (Phase 0 quarantine):
    "Evidence produced during Phase 0 may modify the experimental instrument only.
     Phase 0 evidence MUST NOT strengthen, weaken, validate, or falsify any Nidaa
     assumption, hypothesis, canonical claim, OQ, or evidence ledger state. Its
     only legitimate outcome is improving the quality and runnability of the
     experiment."
  Exit gate: instrument runnable (all templates completable, metrics collectable,
             pressure protocol works) -> proceed to Phase 1. Instrument broken ->
             revise and re-run Phase 0. This is an INSTRUMENT result, never a
             hypothesis result.

PHASE 1 — PROXY PILOT (after Phase 0 instrument-validated)
  Design: docs/Nidaa-Day12-Pilot-Definition.md (within-subject A/B; 21 days;
          6–12 real non-humanitarian participants; one shared doc; 5 workflow
          actions; zero product).
  Purpose (mapping): TEST THE MECHANISM (proxy). Tests A6b canonical (W-C)
           mechanics + A8 economics, in an accessible proxy.
  Purpose (narrative): remove obvious design flaws before approaching external
           orgs; get a real falsification read on maintainability and cost/value.
  Exit gate: pre-registered thresholds in Day 12 def §F. Any falsify = stop and
             write up. Any strengthen = license Phase 2 per the CONDITIONAL SECOND-
             PROXY gate below (not automatic; not a claim).
  QUALIFYING EXIT (post-red-team R2, CONDITIONAL): a second proxy is RECOMMENDED
             (different group + ideally non-founder steward) to separate the founder
             effect from the mechanism, but is REQUIRED only if the first proxy is
             ambiguous or shows strong founder effects. If the first proxy is strong,
             well-documented, AND an NGO is willing to collaborate on a limited
             observational field trial, proceeding to Phase 2 is reasonable while
             clearly acknowledging limitations. A Phase-2 NGO field trial is ITSELF
             an experiment, not a validation declaration.

PHASE 2 — NGO / VOLUNTEER FIELD TRIAL (AFTER Phase 1 strengthens)
  Trigger: Phase 1 strengthened A6b AND A8 (per §F/H of Day 12 def).
  Purpose (mapping): TEST THE MECHANISM INSIDE A REAL ORGANIZATION.
  Setting: ONE small NGO or volunteer organization (NOT a large NGO deployment —
           respects the standing constraint). Real operational workflow.
  Framing (APPROVED outreach script, director, this session — verbatim):
    "We've run a controlled pilot that suggests this coordination approach may
     reduce friction. We'd like to observe whether the same mechanism is useful
     in your existing workflow — not replace your tools."
  What changes vs Phase 1: a REAL locally-legitimate coordinator; real
    verification/trust dynamics; this is where Day 11 H2/H3 variables become
    observable (see §3). Still an observation experiment, not a product sale.
  Exit gate: does the mechanism hold in a real org's workflow under real burden?
    If yes → Phase 3 candidate. If no → the proxy over-optimized; write up.

PHASE 3 — HUMANITARIAN / DISASTER-RESPONSE VALIDATION (LAST, CONDITIONAL)
  Trigger: ONLY after Phase 2 provides encouraging real-org evidence.
  Purpose (mapping): EVALUATE HUMANITARIAN TRANSFERABILITY.
  Setting: humanitarian / disaster-response context (requires access not
    currently held — see constraints).
  Purpose (narrative): finally address OQ-1 (acute→protructed transfer) and OQ-2
    (formal-fails vs coexist) and generate the G1/G2 practitioner cases the
    ledger still lacks.
  Exit gate: real coordinator cases via the six-question gate; convergence
    across independent sources. Until then, Nidaa thesis = UNVALIDATED.

2. CONCURRENCY PRINCIPLE (do NOT wait for Phase 1 to end)
==================================================================
Director directive (this session), adopted as standing process:
  - NOW: run Phase 0 (instrument validation), then Phase 1 (proxy pilot).
  - DURING the proxy: continue building relationships with NGOs / practitioners

proxy cannot be manufactured faster, but the NGO/practitioner pipeline can advance

3. RECONCILIATION — Day 11 vs Day 12 pilot scope
==================================================================
The two pilot artifacts are CONSISTENT but differ in scope. Stated explicitly to
prevent drift into "two pilots":

  DAY 11 pilot def: broader. H1 (W-C maintainability) + H2 (consult BECAUSE
    coordinator locally legitimate / trust-anchor) + H3 (outlives unowned board).
    Allows a role-play/simulation fallback if no real coordinator obtainable.

  DAY 12 pilot def (Phase 1 proxy): DELIBERATELY NARROWER. Tests ONLY:
    - H1 equivalent: W-C maintainability under pressure (within-subject A/B).
    - A8: cost/value economics.
    Explicitly DEFERS: trust-anchor (H2), verification/trust-marker dynamics, and
    local-legitimacy effects. Reason: a benign proxy group has no real
    locally-legitimate coordinator to observe, so those variables would be
    unmeasurable noise in Phase 1.

  RESOLUTION: Day 11's H2/H3 variables are NOT dropped — they move to PHASE 2,
  where a real NGO coordinator's local legitimacy and verification behavior are
  actually observable. Phase 1 stays narrow (mechanics + economics); Phase 2
  widens to trust/verification. This keeps Phase 1 smallest-possible while
  preserving the Day 11 variable set across the roadmap.

  PHASE 0 CARVE-OUT: Phase 0 (instrument validation) is orthogonal to this scope
  split. It tests neither W-C nor W-A nor A8; it rehearses the instrument. It does
  not move, soften, or preempt any Day 11 / Day 12 variable assignment. The split
  above remains the governing scope map for Phases 1–3.

==================================================================
4. CONSTRAINTS HONORED
==================================================================
- No large NGO deployment (Phase 2 = small org only).
- No humanitarian access assumed (Phase 3 gated on access not yet held).
- Limited resources / single founder (within-subject A/B; one doc; 5 actions).
- Evidence generation > feature completeness (zero product in all phases).
- Proxy cannot speak to OQ-1/OQ-2/G1/G2 — those stay OPEN until Phase 3 (or a
  real practitioner case arrives via the pipeline concurrently).

==================================================================
5. WHAT THIS ROADMAP DOES NOT DO
==================================================================
- Does not promise Phase 2 or 3 will happen (both gated on evidence).
- Does not let a Phase-1 "strengthen" leak into humanitarian claims.
- Does not pause the practitioner pipeline during Phase 1.
- Does not promote any assumption to a conclusion at any gate.
- Does not let Phase 0 evidence touch any assumption/hypothesis/ledger state
  (instrument-only, per the Phase 0 quarantine rule).
