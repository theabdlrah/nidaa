# Nidaa — Predictions (baseline before evidence)

**Purpose:** a baseline to prevent hindsight bias. Written before any
conversation. Do NOT edit these priors after evidence arrives — edit the
ASSUMPTION-TRACKER instead. Reality now gets a vote.

Belief = our current probability the assumption is true, pre-evidence.

---

**A1 — Coordination is a priority problem**
- Current belief: 60%
- Why: crises routinely overwhelm comms channels; anecdote + advisor intuition
  say coordination breaks when it matters. But this is untested and may be
  dwarfed by supply/access/security.
- Strengthening: multiple orgs name coordination as a top-3 pain; recent
  incident where coordination failure had a visible cost.
- Weakening: orgs say "coordination is fine, supply/access is the real problem."
- Falsifying: across 5+ conversations, coordination is never raised as a
  priority and is explicitly subordinate to other constraints.

**A2 — Information/matching is a significant bottleneck**
- Current belief: 55%
- Why: private chats fragment info and newcomers can't see it; plausible but
  we haven't confirmed the bottleneck is info vs verification vs trust vs
  logistics.
- Strengthening: orgs describe specific failures to find/connect known help;
  "we didn't know X existed" recurring.
- Weakening: failures trace to supply, access, or physical delivery — not info.
- Falsifying: bottleneck is consistently something else (supply, trust,
  politics) across conversations.

**A3 — Offline capability materially matters**
- Current belief: 65%
- Why: target environments lose connectivity repeatedly by design/circumstance;
  device-first survival seems obviously useful. But may be a nice-to-have
  traded for familiarity.
- Strengthening: orgs report days-long disconnection breaking current tools;
  offline cited as a deciding factor.
- Weakening: disconnection is brief/rare; orgs tolerate it with existing tools.
- Falsifying: offline is never a deciding factor; orgs would not switch for it.

**A4 — Verification materially matters**
- Current belief: 50%
- Why: unverifiable posts are a known harm, but we have no evidence verifier
  trust is the binding constraint vs just having any info at all.
- Strengthening: orgs describe acting on false info with real cost; trusted
  confirmers already exist and are relied upon.
- Weakening: trust is placed in people/sources, not marks; verification adds
  friction without changing behavior.
- Falsifying: verification is seen as irrelevant or a liability in their context.

**A5 — Existing tools are insufficient for at least some communities**
- Current belief: 60%
- Why: WhatsApp/Telegram/Airtable are mature but not purpose-built for
  offline+verifiable coordination; plausible gap for some.
- Strengthening: orgs describe specific unmet needs current tools can't serve.
- Weakening: orgs say current tools are "good enough"; no gap worth a switch.
- Falsifying: every community surveyed finds existing tools sufficient; zero
  expressed need for an alternative.

**A6 — Organizations would trust designated verifiers**
- Current belief: 45%
- Why: trust is local and political; "designated verifier" may not map to who
  communities actually believe. Skeptical baseline.
- Strengthening: orgs name a person/body they'd accept as verifier; existing
  verification roles already function.
- Weakening: no acceptable verifier exists; any designated party is distrusted.
- Falsifying: the very concept of a designated verifier is rejected as
  illegitimate or unsafe in their context.

**A7 — Communities would adopt a new workflow if it solved the problem**
- Current belief: 40%
- Why: adoption is winner-take-most; switching cost is high; strong prior
  skepticism that a new tool clears the threshold even if useful.
- Strengthening: orgs say they'd adopt IF it solved a real pain; concrete
  adoption intent expressed.
- Weakening: orgs say they'd never switch off familiar tools regardless.
- Falsifying: even when the problem is confirmed, no interest in a new workflow.
