# Feasibility Check (operational, not design)

Answer honestly before committing to a start date. A "no" is an OPERATIONAL
constraint, not a design flaw — name exactly what's missing and close it.

1. Proxy group with 6–12 active participants I have legitimate access to?  Y / N
   -> If N: this is the biggest gap. Without a real group, the pilot cannot run.
      Revisit whether a proxy environment is obtainable at all.

2. That group shows fragmented coordination across >=3 channels TODAY?  Y / N
   -> If N: the design's core assumption (real bottleneck) isn't met here.
      Either pick a different group or document that this group lacks the problem.

3. A foreseeable pressure/stress event exists in the next 21 days?  Y / N
   -> If N: the "under pressure" test can't be run. Stage one or pick a window
      that has one (event deadline, request surge).

4. Independent observer recruitable (even a trusted non-expert)?  Y / N
   -> If N: proceed EXPLORATORY (documented downgrade). Not blocking.

   -> If N: A8 cost may exceed capacity; note the cap breach as cost evidence.

6. Can I realistically RUN this in the next 2–4 weeks?  Y / N
   -> If N: state the specific missing item (participants / group / observer /
      time / access) and resolve it before locking a start date.
