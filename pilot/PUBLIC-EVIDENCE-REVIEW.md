# Public Evidence Review — Nidaa Evidence Pyramid (Tier 1)

This file is **Tier 1 (Public Evidence)** of the Evidence Pyramid. It answers the
question *"What is already known?"* before — and alongside — interviews (Tier 2).
Interviews no longer exist to "teach us everything"; their job shifts to
**triangulation**: *"Do the public findings match your reality?"*

Honesty bar (unchanged from ASSUMPTION-TRACKER.md): every row is a REAL, fetchable
source. No report is paraphrased from memory. No citation is invented. Sources marked
`[RETRIEVED]` were fetched and the URL verified live (HTTP 200) this session; sources
marked `[TO RETRIEVE]` are named targets not yet pulled. A row with no source is a gap,
not evidence.

---

## The Evidence Pyramid

| Tier | What | Target | Deliverable | Gate to next tier |
|------|------|--------|-------------|-------------------|
| **1 — Public Evidence** | Reports, evaluations, after-action reviews, datasets, academic literature | 20–30 high-quality sources | Assumption-by-assumption evidence matrix (this file) | — |
| **2 — Field Evidence** | 5–10 substantive conversations (Omar, Shaima, Abdelrahman, iMMAP, referrals) | 5–10 interviews | Information-journey maps; per-assumption confirmation/contriction | Begins in parallel with Tier 1 |
| **3 — Solution Evidence** | Product/design validation | NOT YET | — | Only after **A1, A4, A6 survive** both Tier 1 + Tier 2 |

**Convergence rule:** one report is weak; one interview is weak; **three independent
sources agreeing is strong.** The most valuable finding is where public evidence AND
field evidence say the same thing.

**Phase allocation (now):** ~60% public evidence, ~40% interviews (running in parallel).
The failure modes this avoids: (a) months of outreach before learning what is already
known; (b) months of reading reports and never talking to real people.

## How this feeds Tier 2 (interviews)

Each interview question is now *derived from a public finding*, not open-ended:

- Public: "Humanitarian ops rely heavily on informal WhatsApp networks." → Ask Omar / Shaima: *Is that true in your experience?*
- Public: "Verification becomes difficult during outages." → Ask iMMAP: *Does that match what you see?*
- Public: "Coordination is formalized via clusters but gaps appear under stress." → Ask Abdelrahman / referrals: *Where does it break?*

The interview template (EVIDENCE-INTAKE-TEMPLATE.md) and CURRENT-EVIDENCE-STATE.md remain
the field-side capture. This file is the public-side mirror.
