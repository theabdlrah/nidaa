## Standing processing rule

Every incoming coordinator-level conversation is routed through the **Six A6 Questions** and logged
here via EVIDENCE-INTAKE-TEMPLATE.md discipline (one row per conversation):

1. **Who keeps information?**
2. **How is it kept?** (chat / spreadsheet / notebook / memory / nothing)
3. **What gets written down?** (beneficiary lists / needs / incidents / distributions)
4. **What remains verbal?** (most coordination? decisions? who-has-what?)
5. **What breaks first under pressure?** (surge / insecurity / connectivity loss / burnout)
6. **What is abandoned first?** (the board / verification / updating)

Success criterion: **one real coordinator story** that answers all six — either a SUCCESS case
(structured maintenance that held) or a COLLAPSE case (maintenance dropped under load). Either is
evidence; the milestone is the story, not a report.
