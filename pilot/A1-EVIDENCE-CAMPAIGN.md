# A1 Evidence Campaign — "Does coordination fail as a primary operational pain?"

**Why this file exists.** Per the 2026-07-15 strategic direction, A1
("Coordination is a significant operational problem") is the project's **largest
unresolved risk**. A3 already shows early convergence (public infrastructure evidence
+ Adam interview + Abdelrahman reply). A1 does not. The next milestone is NOT a
prototype — it is *determining whether A1 survives evidence convergence*. If A1 is
falsified, Nidaa must be re-scoped before any design work (Pivot Rule).

This campaign builds the **strongest possible case for OR against A1** — not a
brief for it. Challenge your own conclusion; do not validate it.

## 6. What evidence would STRENGTHEN A1

- A coordinator states coordination is a top-3 pain point.
- A documented incident where a coordination failure had a visible cost.
- Multiple orgs independently rank coordination above logistics/access/security.
- A published evaluation concluding coordination is a systemic bottleneck.

## 7. What evidence would WEAKEN A1

- Orgs consistently rank coordination below supply / access / security.
- *"Coordination is fine; X is the real problem."*
- Mature cluster system shown to absorb load without breakdown in the contexts studied.

## 8. What evidence would FALSIFY A1

Per the Pivot Rule, A1 is **strongly falsified** when:
- Across **3+ independent coordinator interviews** AND **2+ evaluations**, coordination is
  **NOT** a primary problem; existing mechanisms work; no failure with cost is identified.

Consequence if falsified:
- If A1 false **and** A2/A5 also weak → the core thesis (a coordination/information tool is
  needed) collapses → must **pivot / narrow scope / stop** before any further design.
- A1 false but A2/A3/A5 strong → re-scope: the problem is *information access during
  disruptions*, not *coordination failure*; Nidaa's coordinator-led framing must shrink.
