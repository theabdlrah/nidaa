# Evidence Intake Template

Use this template for EVERY piece of evidence collected — a reply, a quote, a
screenshot, a document, a referral. One entry per piece of evidence. The goal is
to capture *what actually happened*, not impressions.

Honesty bar (from ASSUMPTION-TRACKER.md): do NOT write "they liked it",
"positive conversation", or "interesting". Write what was said/observed and what
it does to an assumption.

Copy the block below and fill it in.

---

### Evidence Intake — <DATE>

- **Contact:** (name / org / role)
- **Role:** (coordinator / resident / field ops / program / IMO / volunteer / other)
- **Date:** (YYYY-MM-DD)

- **Exact story / quote:** (verbatim if possible; note if paraphrased. Who said it, in what words.)
- **Information origin:** (where did the information come from — the person, a doc, a screenshot, observed behavior?)
- **Information path:** (how did it reach us — LinkedIn reply, email, referral, forwarded message?)
- **Information consumer:** (who uses this information — beneficiaries, coordinators, orgs, donors?)

- **Bottleneck:** (was the bottleneck information, verification, coordination, logistics, supply, trust, connectivity, or something else?)
- **Workaround / adaptation:** (what did they do when the normal channel failed? e.g. during an outage)

- **Assumptions affected:** (list: A1 / A2 / A3 / A4 / A5 / A6 / A7 — and any failure mode F1–F5)
- **Evidence strength:** (Strong / Moderate / Weak / Anecdotal — based on how direct and verifiable)
- **Strengthens / Weakens / Falsifies:** (for each assumption touched: does this evidence strengthen, weaken, or falsify it? State which.)

---

Notes:
- One contact may produce several evidence intakes (one per distinct claim/event).
- A reply is NOT automatically evidence — only the concrete signal it contains counts.
- File the filled template into `assumption-log/<ORG>-<DATE>.md` after a full conversation,
  and surface the key line in CURRENT-EVIDENCE-STATE.md.
